# Documentation Generator

This skill helps generate comprehensive documentation for code, APIs, and projects.

## When to Use

Use this skill when:
- You need to document code, APIs, or projects
- You want to create README files or technical documentation
- You need to generate API documentation from code
- You want to improve code maintainability through documentation

## What It Does

- Analyzes code to extract documentation needs
- Generates clear, comprehensive documentation
- Creates API documentation from function signatures and comments
- Produces README files with project overview, usage, and examples
- Formats documentation in Markdown or other standard formats

## How to Use

1. Specify what you want to document (code, API, project)
2. Provide the code or files to document
3. Indicate the desired format (Markdown, HTML, etc.)
4. The assistant will generate comprehensive documentation
5. Review and refine the generated documentation as needed

## Best Practices

- Keep documentation close to the code it describes
- Use clear, concise language
- Include examples for complex functionality
- Document both what the code does and why
- Update documentation when code changes
- Follow established documentation standards for your language/project
